  • Angular 2


    Angular 2 é um framework mantido pelo Google que auxilia na abstração e manipulação do DOM da lógica do aplicativo, a abstrair o acoplamento entre o lado cliente e o lado servidor da aplicação e a melhorar a qualidade dos testes.
    Angular é basicamente um conjunto de bibliotecas (algumas core, outras não) que trabalham em cooperação.
    O framework segue o padrão MVC (Model View Controller) e presa pelo baixo acoplamento entre as camadas de apresentação, dados e componentes lógicos. Utilizando-se de injeção de dependência ele traz alguns componentes que normalmento são utilizados no lador servidor para o lado cliente deixando, consequentemente, o backend mais leve.
    Abaixo segue a arquitetura básica do Angular 2:

    Aplicações Angular são desenvolvidas pela composição de templates HTML enriquecido com anotações e marcações angular. Para manipular os templates escrevemos componentes e criamos serviços para executar a lógica da aplicação. Componentes e serviços são emapcotados em módulos.
    Componentes nada mais são que uma classe com a anotação @Componente, exemplo de componente: 
    @Component({
						selector:'hero-list',
						templateUrl:'app/hero-list.component.html',
						providers:   [ HeroService ]
					})
					export class HeroListComponent implements OnInit {
					/* . . . */
					}

    Para saber mais acesse: Angular 2.

  • AngularFire 2


    AngularFire é uma biblioteca de código livre mantida pelo time da Firebase e sua comunidade de desenvolvedores.
    Ele provê uma associação tri-direcional, ou seja, ocorrendo uma alteração na view, ou no objeto de dados do Angular, ou no banco de dados de tempo real do Firebase reflete automaticamente nos demais.

    Para saber mais acesse: AngularFire 2.

  • Application Shell.


    Uma aplicação shell é o conjunto de HTML, CSS e Javascript mínimos necessários para "carregar" a interface de uma aplicação web progressiva e um dos componentes que garantem uma boa performance da aplicação. A primeira vez que a aplicação carrega ela é imediatamente cacheada, isto é, o shell não precisa ser carregado toda vez, apenas o conteúdo necessário será carregado.
    A arquitetura de uma aplicação shell separa a infraestrutura da aplicação principal da interface e dos dados. Toda a interface e a infraestrutura são cacheadas localmente utilizando-se um servivce worker para que posteriormente a aplicação seja carregada mais rápida.

    Para saber mais acesse: Application Shell.

  • Firebase 3


    Firebase é uma plataforma móvel unificada BaaS (Backend as a Service).Ela fornece ao desenvolvedor diversos serviços baseado em nuvem. Alguns dos serviços oferecidos pelo Firebase são:
    • Análise: Ferramenta que auxília a entender o comportamento do usuário, e estáticas de requisições ao aplicativo para auxíliar na tomada de decisões;
    • Armazenamento: Possibilita guardar qualquer tipo de dados (Arquivos, fotos, vídeos...);
    • Autenticação: Autenticação integrada com as principais redes sociais (Facebook, GitHub, ...);
    • Banco de dados em tempo real: Significa que qualquer alteração realizada no banco é refletida em todas as aplicações conectadas. O banco de daos é NoSQL e os dados são armazenados como objetos JSON;
    • Hospedagem: Permite fazer o deploy da aplicação tornando-a acessível via web (HTTP/HTTPS), oferece ferramenta para controle de versão, escalonamento de máquinas para acompanhar o volume de requisições, etc..;
    • Notificações:Permite o envio de notificações direcionadas aos usuários.


    Para saber mais acesse: Firebase 3.

  • Ionic 2


    Ionic framework é uma SDK (Software Development Kit) de código aberto que possibilita ao desenvolvedor de aplicações web, que já possui conhecimento em HTML5, CSS e JavaScript, a criar aplicações mobile.
    Ionic foca principalmente na interação com a interface do aplicativo, ele visa simplificar o desenvolvimento do front-end. Basicamente ele preenche a lacuna do HTML5 para o desenvolvimento nativo do aplicativo.
    Além do framework permitir que aplicações mobile sejam criadas praticamente do mesmo modo de uma aplicação web comum ainda permite que os testes sejam realizados diretamente pelo browser.

    Para saber mais acesse: Ionic 2.

  • Progressive Web Application


    Numa combinação entre aplicações web e aplicações nativas obtêm-se as aplicações web progressivas, este tipo de aplicação possui um carregamento rápido mesmo em redes instáveis, contém também o envio de notificações relevantes. Inicialmente, o termo aplicações web progressivas (progessive web application) foi utilizado como definição para os aplicativos web que empregariam novas tecnologias.

    Através de uma URL (Uniform Resource Locator) o usuário acessa o aplicativo, assim como realiza o acesso a uma página comum da internet, isto é, um website. Ao realizar repetidas visitas ao site e uma maior interação com o mesmo, o usuário recebe uma mensagem para instalação do aplicativo, tornando-o a partir de então, um aplicativo com características nativas em seu aparelho móvel. Tornando totalmente dispensável o acesso à lojas de aplicativos. Sob o ponto de vista dos desenvolvedores, mantém-se somente um código fonte, pois o conteúdo é interpretado pelos browsers de plataformas diversas.



    Para saber mais acesse: Progressive Web Application.

  • Service Workers


    É um script executado pelo browser em background, separado de uma página web, disponibilizando recursos que não precisam de uma página web ou de interação com o usuário.
    Desenvolvedores podem registrar um service worker para interceptar todas as requisições feitas ao servidor para criar cache local e entregar o conteúdo armazenado localmente no caso da rede não estar disponível, isto é, possibilita que um aplicativo web móvel funcione plenamente mesmo estando off line.

    Para saber mais acesse: Service Workers.

  • TypeScript


    TypeScript é uma linguagem de programação de código aberto criada e mantida pela Microsoft, é um "superconjunto" do JavaScript criada para desenvolvimento JavaScript em larga escala.
    Essa linguagem permite ao desenvolvedor escrever código utilizando uma estrutura fortemente tipada, benefícios como:
    • Criação de Interfaces
    • Criação de Classes
    • Criação de módulos (facilita a exportação do código)

    O código TypeScript é compilado em JavaScript puro.

    Para saber mais acesse: TypeScript.

  • Web App Manifest


    Trata-se de um arquivo que provê meta dados úteis sobre uma aplicação web como por exemplo o nome da aplicação, ícones, configurações de inicialização entre outras.
    É um arquivo JSON que descreve alguns aspectos superficiais sobre como a aplicação web deve ser inicializada permitindo uma experiência mais próxima de uma aplicação nativa.

    Para saber mais acesse: Web App Manifest.
